# 3 分钟 Demo 演示稿

本演示稿用于面试或作品集展示。目标是在 3 分钟内说明用户问题、核心 Workflow、技术取舍和产品闭环，不逐页介绍所有字段。

## 演示前准备

- 在本地浏览器预先准备一套完整 `AgentState`
- 建议目标岗位使用“AI 产品经理”或与面试岗位一致的角色
- 提前上传一份可公开展示的 JD PDF
- 保证 JobProfile、AbilityProfile、Gap、Roadmap、Projects、Resume、Interview 和 Tasks 已生成
- 至少完成一项任务，便于 Dashboard 展示真实进度变化
- 保留 API Key 正常配置，但演示时优先展示已有结果，避免等待外部请求

## 时间轴

| 时间 | 页面与操作 | 展示内容 | 要传达的价值 |
| --- | --- | --- | --- |
| 0:00–0:20 | 打开首页 `/` | 目标岗位、主链路进度、能力/简历/面试指标、下一步推荐 | 这是围绕一个目标岗位持续工作的求职准备 Agent，不是单次聊天工具 |
| 0:20–0:40 | 点击“调整求职目标”，再进入 `/upload` | 目标岗位、目标公司、JD PDF 上传入口 | 使用真实 JD 作为后续分析的共同上下文 |
| 0:40–1:00 | 进入 `/analysis` 或 `/analysis-jd` | 职责、必备技能、工具、关键词和评估标准 | LLM 将复杂 JD 转换为结构化 JobProfile |
| 1:00–1:20 | 进入 `/assessment`，随后打开 `/gap` | 用户经历与技能、匹配度、缺失技能、优先差距 | 将“岗位需要什么”和“用户具备什么”进行对比 |
| 1:20–1:45 | 依次打开 `/roadmap` 和 `/projects` | 90 天阶段目标、每日任务、三档项目和交付物 | 从分析结果直接生成学习和作品集行动 |

| 2:40–2:55 | 返回首页 `/`，再打开 `/profile` | 任务进度变化、成长指标、下一步推荐 | Tasks 与 Dashboard/Profile 形成反馈闭环 |
| 2:55–3:00 | 停留在 Profile 或首页 | 简述技术与边界 | 当前是 localStorage MVP，不是完整生产系统 |

## 推荐讲解话术

### 0:00–0:20：定位

“职途 Agent 解决的是求职准备过程割裂的问题。用户通常分别做 JD 分析、学习规划、项目、简历和面试，但这些步骤没有共享上下文。这个项目把它们组织成一条连续 Workflow，并在首页展示真实进度和下一步。”

### 0:20–0:40：真实输入

“用户先确定目标岗位，也可以上传真实 JD。PDF 会在浏览器中提取全文，岗位分析和后续能力判断都围绕同一个岗位上下文进行。”

### 0:40–1:00：岗位画像

“这里不是直接展示一段模型回答，而是生成结构化 JobProfile，包括职责、必备技能、工具、关键词和招聘评估标准，后续节点可以继续复用。”

### 1:00–1:20：能力与差距

“用户补充教育、经历和技能后生成 AbilityProfile。Gap 不再额外调用模型，而是用规则对比岗位画像和能力画像，输出优先差距和行动建议，让结果更稳定、可测试。”

### 1:20–1:45：行动规划

“差距会被拆成 90 天路线，再生成入门、进阶和面试亮点三档项目。每个项目包含目标能力、功能、技术栈、交付物和面试讲法，重点是让建议可以真正执行并写进作品集。”

### 1:45–2:25：求职表达

“简历和面试继续使用前面的岗位、Gap 和项目数据。简历报告强调关键词和项目表达；模拟面试覆盖岗位理解、项目、AI 产品设计、技术和行为题，并支持回答评分与优化。”

### 2:25–2:55：执行闭环

“Roadmap、项目交付物和面试建议会汇总成 Tasks。任务状态变化后，Dashboard 和 Profile 的进度会同步更新，从建议生成走到执行反馈。”

### 2:55–3:00：边界

“当前版本是 MVP，数据使用 localStorage，没有登录和数据库。RAG 也是 PDF 全文上下文问答原型，不是向量数据库方案。下一步会先做用户验证，再决定是否引入云端数据和完整 RAG。”

## 技术追问时的补充点

- React 19、TypeScript、TanStack Router / Start
- `AgentState` 串联全部结构化结果
- SiliconFlow 与 Tavily 通过 server functions 调用，Key 不进入客户端
- Gap、Roadmap、Tasks 和 Dashboard 指标采用本地规则
- 项目、简历和面试部分具备 fallback
- PDF 使用 PDF.js 在浏览器端提取文本

## 时间不足时的 90 秒版本

只展示四个页面：

1. 首页：定位和完整进度
2. Gap：证明个性化诊断
3. Projects / Resume：证明结果可以转化为作品集和求职表达
4. Tasks / Dashboard：证明形成执行反馈闭环

## 外部 API 异常预案

- 使用预先保存在 `localStorage` 中的完整演示数据
- 不在现场重新生成全部报告
- 如需要说明 fallback，展示已有结果并解释部分模块在模型失败时会返回规则生成内容
- 不宣称系统完全离线可用，因为岗位画像与能力画像仍依赖有效的 AI 请求
